# Teaching Notes

- The learner built most of the project through vibe coding and currently understands the system at a high level.
- Teach in an interview-first sequence: concise answer, mental model, code trace, adversarial follow-ups, retrieval practice.
- Every generic Java or Agent topic should be connected to a concrete PaperLoom file or runtime decision when possible.
- Be strict about the difference between "I can recognize this" and "I can explain or derive this without prompts."
- Keep answers honest: emphasize ownership of problem framing, verification, tradeoffs, and evolution rather than claiming manual authorship of every line.
