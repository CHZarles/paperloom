# 关于这段实践

我是 Charles。PaperLoom 是我持续学习 AI 系统工程的主项目。我从一个能运行的 RAG 产品开始，
一路处理 PDF 建模、`harness_py` 工具、Evidence、Evaluation、前端和产品边界。很多判断是在真实链路
跑起来以后才变清楚的。

这个仓库也是我的公开工程档案。代码要能运行，架构主张和实验数字要能找到依据。我会保留那些
改过系统方向的失败：为什么当时觉得方案可行，数据后来怎样推翻它，我又删掉了什么。几年后再看，
这里应该能看出技术能力和判断方式是怎样一起变化的。

## 目前关注的方向

- Evidence-bounded Agentic RAG
- Research Agent 和工具协议
- PDF Reading Model
- Lexical Retrieval、Dense Retrieval 与 Reranking Evaluation
- Trace、Golden Data 与 LLM Judge 校准
- 用可观察 Tool Trajectory 优化和蒸馏本地模型
- 面向复杂工作流的前端体验
- AI-assisted software engineering
