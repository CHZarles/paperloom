The PaperLoom Project Spine

Your first interview win is not remembering 293 Java files. It is holding one correct mental model that every deeper answer can attach to.

The One-Sentence Model

PaperLoom is an evidence-bounded Agentic RAG workbench for research PDFs: Java fixes the user's authorized paper scope and durable product state, Python runs a tool-using research Agent inside that scope, and an answer is accepted only when its claims cite exact locations read from the canonical Reading Model.

This sentence contains four interview-grade ideas:

  • Product: a research-paper reading workbench, not a generic chatbot.
  • Boundary: Java owns authorization and durable state; Python owns bounded research behavior.
  • Agent: retrieval is a model-directed sequence of tool calls, not one search followed by one generation.
  • Invariant: citeable claims must trace to exact content that the Agent was authorized to read.

The Live Path

Do not describe this as "Java calls Python, then Python calls the LLM." That is true but too shallow. The meaningful story is the movement of authority and evidence.

The Defensible Two-Minute Answer

Problem. Typical RAG demos optimize for plausible answers, but a paper-reading product also needs every claim to reopen the supporting source location.

Design. PaperLoom converts PDFs into a product-owned Reading Model with pages, sections, typed elements, locations, provenance, and visual assets. Java locks the user's paper scope and owns persistent state. A Python Agents SDK runtime performs research with narrow tools inside that scope.

Control. Search results are only candidates. The Agent must disclose a paper, find a location, and read that exact location before an Evidence ID exists. Final submission is validated against the Evidence Ledger and coverage rules.

Storage. MySQL is canonical, Qdrant is a rebuildable sparse candidate index, and MinIO retains original and visual artifacts. Java converts accepted evidence into durable references so old citations can reopen.

Tradeoff. This adds orchestration and validation complexity, but it makes authorization and citation behavior deterministic rather than trusting model prose.

Retrieval Drill

Answer aloud before opening each panel. Aim for 20 to 40 seconds per question.

1. Why is this Agentic RAG rather than ordinary RAG?

The model chooses a sequence of actions inside one run: discover papers, resolve identity, search locations, reformulate, read exact content, consult a method guide, ask for clarification, abstain, or submit. Deterministic tools still constrain authorization, evidence creation, and final acceptance.

2. Why can Qdrant search output not be cited directly?

Qdrant is a rebuildable navigation index whose payload is not canonical truth. Citeable evidence is created only when Java revalidates the authorized scope and Current READY model, then reads the exact canonical location from MySQL.

3. Why split Java and Python?

Java holds hard product boundaries and durable state, while Python provides the fast-moving Agent runtime and tool loop. The split prevents model orchestration code from deciding its own global authorization scope, though it adds a network boundary and contract-management cost.

4. What is the system's central invariant?

A paper-content claim in an accepted answer must remain connected to exact evidence read from a paper that the current conversation was authorized to use.

5. What would you improve first?

A strong answer should start from measurement: held-out Candidate, Read, and Cited recall; lexical-miss analysis; large-scope filter latency; capacity; and failure classification. Only then justify a reranker, dense retrieval, or another storage change.

Interview Honesty

Do not say, "AI wrote it, so I do not know." Also do not claim intimate authorship you cannot defend.

Use: "I used AI heavily for implementation, but I owned the product constraints, selected and verified the architecture, built tests and evaluation gates, and iterated based on observed failures. I can walk you through the live path and the tradeoffs." Then prove it.
