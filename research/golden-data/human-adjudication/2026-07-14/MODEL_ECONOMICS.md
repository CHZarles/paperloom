# GPT-5.5 与 MiniMax-M3 离线投入产出对比

分析日期：2026-07-15

本页只读取已经保存的 30 Case 单路径运行和冻结人工标签，不调用模型，也不修改生产配置。
这里的“投入产出”只覆盖已保存的 Token、调用次数、累计耗时和人工质量；供应商账单价格没有保存在
Run 中，因此不能伪造精确的人民币或美元成本。

## 对比结果

| 指标 | GPT-5.5 | MiniMax-M3 | 解释 |
| --- | ---: | ---: | --- |
| 人工总体通过 | `28/30 = 93.3%` | `22/30 = 73.3%` | GPT-5.5 多通过 6 个 Case。 |
| 事实依据通过 | `28/28 = 100.0%` | `18/25 = 72.0%` | 排除不适用 Case。 |
| 盲审偏好 | `24/30` | `4/30` | 另有 2 个平局。 |
| 模型调用 | `159` | `212` | GPT-5.5 少 `25.0%`。 |
| 工具调用 | `173` | `167` | GPT-5.5 多 6 次，但没有带来更长总耗时。 |
| Prompt Token | `1,042,828` | `2,589,747` | MiniMax 数字缺少技术失败 Case。 |
| Completion Token | `27,114` | `112,682` | MiniMax 数字缺少技术失败 Case。 |
| 已记录总 Token | `1,069,942` | `2,702,429` | GPT-5.5 只使用 MiniMax 已记录量的 `39.6%`。 |
| 累计 Case 耗时 | `1,696.2 秒`（约 `28.3` 分钟） | `2,165.4 秒`（约 `36.1` 分钟） | MiniMax 技术失败 Case 没有保存 duration，差距被低估。 |
| 冻结运行技术失败 | `0/30` | `1/30` | GPT 首轮有一次传输超时，只按既定规则重跑该技术失败。 |
| 每个人工通过所需 Token | 约 `38,212` | 至少约 `122,838` | MiniMax 每个通过答案约为 GPT-5.5 的 `3.21` 倍。 |

Token 和耗时来自每个 `harness_run.json` 的 `diagnostics` 求和。MiniMax 的
`mint_vs_tau_interaction_comparison_001` 以技术失败结束，没有完整的 Token、模型调用和耗时统计，
所以表中的 MiniMax 投入是下界，不是上界。

## 如何补上真实价格

保存产物没有记录两个实际 Endpoint 的单价：GPT-5.5 通过 `codex` Responses Provider，MiniMax-M3
通过 MiniMax Chat Completions Provider。公开价格不一定等于这两个账号或代理 Endpoint 的实际
结算价，因此本页只给出可复算公式。

设每百万输入、输出 Token 的实际结算价分别为 `G_in`、`G_out`、`M_in`、`M_out`，且使用同一种
货币，则：

```text
GPT-5.5 运行成本 = 1.042828 * G_in + 0.027114 * G_out
MiniMax 运行成本 >= 2.589747 * M_in + 0.112682 * M_out
```

如果先用一个混合 Token 单价做粗略决策：

- GPT-5.5 的混合单价不超过 MiniMax 的约 `2.53` 倍时，完整 30 Case 运行成本不会更高；
- 按“每个人工通过答案”计算，GPT-5.5 的混合单价不超过 MiniMax 的约 `3.21` 倍时，质量调整后的
  成本不会更高；
- MiniMax 有一个未计入 Token 的技术失败，因此这两个阈值都偏保守。

## 决策建议

基于当前证据，GPT-5.5 应作为**质量基线和默认候选**。它同时取得更高人工通过率、更完整的事实
依据、更少模型调用、更少 Token 和更短累计耗时。当前没有证据支持为了提升质量继续修改检索器。

但本页不直接修改生产默认模型。最终切换还需要两个外部事实：

1. 两个实际 Endpoint 的账单输入/输出单价；
2. 生产环境的并发限制、限流、稳定性和数据合规要求。

如果这些条件允许，应把 GPT-5.5 设为默认研究模型，并把 MiniMax-M3 保留为成本或可用性备选。
如果业务必须使用 MiniMax，下一轮实验只针对 7 个已确认的事实依据失败，验证一个通用改进；在
相关 Case 和稳定回归集都改善前，不进行全量 30 Case 重跑。
