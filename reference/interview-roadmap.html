The course moves from a defensible project story to implementation depth, then mixes topics so recall survives interview pressure.

Default Sequence

  1. Build the project spine and a two-minute explanation.
  2. Trace one request end to end in real code.
  3. Learn Java and Spring concepts at the point where the trace needs them.
  4. Deep-dive the Agent loop, tool protocol, RAG, and evidence validation.
  5. Study storage and distributed-system tradeoffs through actual design decisions.
  6. Convert experiments, failures, and tests into behavioral interview stories.
  7. Interleave Java trivia, code reading, system design, and hostile follow-ups.
Mastery rule: a topic is not learned when it looks familiar. It is learned when you can explain it without notes, locate it in the repository, predict a failure mode, and answer one counterfactual design question.
